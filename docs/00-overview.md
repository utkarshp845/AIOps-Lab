# Overview

AIOps Lab teaches AI infrastructure by starting with the parts every production engineer already understands: services, logs, health checks, metrics, containers, and operational debugging.

The AI part is introduced as an assistant that observes a service, not as a giant platform on Day 1.

## Learning Order

1. Run a local app.
2. Generate realistic service signals.
3. Read logs and metrics.
4. Ask an AI SRE Assistant for evidence-grounded analysis.
5. Add observability practices.
6. Containerize the workflow.
7. Move toward Kubernetes.
8. Discuss production constraints such as security, cost, evals, and model serving.

## What Day 1 Includes

- A FastAPI demo service.
- Structured JSON logs.
- Prometheus-style metrics.
- A deterministic AI SRE Assistant fallback.
- Optional OpenAI-compatible LLM integration.
- Docker Compose.
- Tests and CI.
